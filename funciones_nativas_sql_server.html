Clase 01 · SQL Server

Funciones Nativas
de SQL Server

Herramientas del lenguaje para manipular cadenas de texto, trabajar con fechas, convertir tipos de datos y operar con números, directamente dentro de tus consultas.

¿Qué son las funciones nativas?

SQL Server incluye cientos de funciones predefinidas que podés usar directamente en tus consultas sin escribir lógica adicional. Están agrupadas por el tipo de dato con el que operan.

Una función nativa recibe uno o más argumentos, realiza una operación, y devuelve un valor resultado. Se pueden usar en cualquier lugar donde pondría una expresión: en SELECT, WHERE, ORDER BY, CASE, e incluso dentro de otras funciones.
Cadenas TEXT
Manipulación de texto: recortar, reemplazar, cambiar mayúsculas, buscar subcadenas, concatenar.
SUBSTRING · LEN · UPPER · LOWER
TRIM · REPLACE · CHARINDEX · CONCAT
Fechas DATE
Obtener la fecha actual, extraer partes, calcular diferencias, formatear y desplazar fechas.
GETDATE · YEAR · MONTH · DAY
DATEDIFF · DATEADD · FORMAT · EOMONTH
Conversión CAST
Cambiar el tipo de dato de un valor: número a texto, texto a fecha, entero a decimal y más.
CAST · CONVERT · TRY_CAST · TRY_CONVERT
ISNULL · COALESCE
Números NUM
Redondear, truncar, obtener valor absoluto, potencias, raíces y generación de números aleatorios.
ROUND · FLOOR · CEILING · ABS
POWER · SQRT · RAND

Cómo se usan

ejemplo_general.sql 
-- En SELECT: transformar valores al mostrarlos
SELECT
    UPPER(Apellido)                    AS ApellidoMayus,
    LEN(Nombre)                        AS LargoNombre,
    YEAR(GETDATE()) - YEAR(FechaNac)    AS EdadAprox,
    ISNULL(Telefono, 'Sin teléfono')  AS Contacto
FROM Empleados;

-- En WHERE: filtrar usando el resultado de una función
SELECT Nombre, FechaNac
FROM   Empleados
WHERE  MONTH(FechaNac) = MONTH(GETDATE()); -- nacidos este mes

-- Funciones anidadas: resultado de una como argumento de otra
SELECT UPPER(TRIM(Nombre)) AS NombreLimpio
FROM   Empleados;
💡
Las funciones no modifican los datos almacenados. Solo transforman los valores al momento de mostrarlos o compararlos. Para modificar datos en la tabla hay que usar UPDATE.

② Referencia de funciones

UPPER / LOWERCadena
Convierte todos los caracteres a mayúsculas o minúsculas.
UPPER(expresion)
LOWER(expresion)
UPPER('hola') → 'HOLA'
LOWER('MUNDO') → 'mundo'
LENCadena
Devuelve la cantidad de caracteres de una cadena, sin contar los espacios finales.
LEN(expresion)
LEN('SQL') → 3
LEN('Hola ') → 4
SUBSTRINGCadena
Extrae una porción de texto, indicando posición de inicio y cantidad de caracteres. La posición empieza en 1.
SUBSTRING(cadena, inicio, largo)
SUBSTRING('SQL Server', 5, 6) → 'Server'
TRIM / LTRIM / RTRIMCadena
Elimina espacios. TRIM elimina de ambos extremos, LTRIM del inicio, RTRIM del final.
TRIM(cadena)
LTRIM(cadena) / RTRIM(cadena)
TRIM(' hola ') → 'hola'
REPLACECadena
Reemplaza todas las ocurrencias de una subcadena por otra.
REPLACE(cadena, buscar, reemplazo)
REPLACE('a-b-c', '-', '/') → 'a/b/c'
CHARINDEXCadena
Devuelve la posición de la primera aparición de una subcadena. Retorna 0 si no la encuentra.
CHARINDEX(buscar, cadena)
CHARINDEX('@','user@mail.com') → 5
CONCATCadena
Une dos o más cadenas. Ignora NULLs automáticamente (a diferencia del operador +).
CONCAT(val1, val2, ...)
CONCAT('Juan', ' ', 'Pérez') → 'Juan Pérez'
LEFT / RIGHTCadena
Extrae N caracteres desde la izquierda o la derecha de la cadena.
LEFT(cadena, n)
RIGHT(cadena, n)
LEFT('SQL Server', 3) → 'SQL'
RIGHT('SQL Server', 6) → 'Server'
REPLICATECadena
Repite una cadena N veces. Útil para formatear o rellenar.
REPLICATE(cadena, n)
REPLICATE('AB', 3) → 'ABABAB'
REVERSECadena
Invierte el orden de los caracteres de una cadena.
REVERSE(cadena)
REVERSE('SQL') → 'LQS'
STRCadena
Convierte un número a cadena de texto con longitud y decimales definidos.
STR(numero, largo, decimales)
STR(3.1415, 6, 2) → ' 3.14'
STUFFCadena
Elimina un segmento de texto e inserta otro en su lugar.
STUFF(cadena, inicio, largo, nuevo)
STUFF('ABCDE', 2, 2, 'XY') → 'AXYDE'

③ Caso práctico — construir nombre completo y extraer dominio de email

cadenas_practico.sql 
SELECT
    -- Nombre completo con formato "APELLIDO, Nombre"
    CONCAT(UPPER(Apellido), ', ', UPPER(LEFT(Nombre,1)), LOWER(SUBSTRING(Nombre,2,100)))
        AS NombreFormato,

    -- Largo del apellido
    LEN(Apellido) AS LargoApellido,

    -- Email sin espacios y en minúsculas
    LOWER(TRIM(Email)) AS EmailLimpio,

    -- Dominio del email (todo lo que va después del @)
    SUBSTRING(
        Email,
        CHARINDEX('@', Email) + 1,
        LEN(Email)
    ) AS Dominio,

    -- Reemplazar guiones del teléfono
    REPLACE(Telefono, '-', '') AS TelefonoNumerico

FROM Empleados;
⚠️
La concatenación con el operador + devuelve NULL si cualquiera de los valores es NULL. Usá siempre CONCAT() que ignora los NULLs, o combiná con ISNULL() para protegerte.

Funciones de fechas

Las fechas son uno de los tipos de datos más usados y más complejos. SQL Server tiene funciones para obtener la fecha actual, extraer partes, calcular diferencias y formatear la salida.

② Fecha y hora actual

FunciónDescripciónEjemplo de resultado
GETDATE()Fecha y hora actual del servidor (sin zona horaria)2025-06-15 14:32:00.000
SYSDATETIME()Fecha y hora con más precisión (nanosegundos)2025-06-15 14:32:00.1234567

③ Extraer partes de una fecha

partes_fecha.sql 
SELECT
    FechaNac,
    YEAR(FechaNac)        AS Anio,       -- 1990
    MONTH(FechaNac)       AS Mes,        -- 6
    DAY(FechaNac)         AS Dia,        -- 15
    DATEPART(week,   FechaNac) AS Semana,   -- número de semana del año
    DATEPART(weekday,FechaNac) AS DiaSemana, -- 1=Dom ... 7=Sáb
    DATENAME(month,  FechaNac) AS NombreMes  -- 'June' (según idioma)
FROM Empleados;
💡
DATEPART devuelve un número entero. DATENAME devuelve el nombre como texto (en el idioma configurado en el servidor). Para fechas en español usá FORMAT(), que se ve en el punto siguiente.

④ DATEDIFF — diferencia entre fechas

datediff.sql — calcular edades y antigüedad 
SELECT
    Nombre,
    FechaNac,
    FechaIngreso,

    -- Edad en años (resta el año si todavía no llegó el cumpleaños)
        DATEDIFF(year, FechaNac, GETDATE())
    - CASE
        WHEN MONTH(FechaNac) > MONTH(GETDATE()) THEN 1
        WHEN MONTH(FechaNac) = MONTH(GETDATE())
          AND DAY(FechaNac) > DAY(GETDATE()) THEN 1
        ELSE 0
      END                                      AS EdadExacta,

    DATEDIFF(month, FechaIngreso, GETDATE()) AS MesesEnEmpresa,
    DATEDIFF(day,   FechaIngreso, GETDATE()) AS DiasEnEmpresa

FROM Empleados;

⑤ DATEADD — sumar o restar a una fecha

dateadd.sql 
SELECT
    GETDATE()                              AS Hoy,
    DATEADD(day,    30,  GETDATE())        AS En30Dias,
    DATEADD(month, -3,  GETDATE())        AS Hace3Meses,
    DATEADD(year,   1,  GETDATE())        AS AnioQueViene,
    EOMONTH(GETDATE())                    AS FinDeEsteMes,
    EOMONTH(GETDATE(), -1)               AS FinMesAnterior;

-- Partes válidas para DATEADD/DATEDIFF:
-- year · quarter · month · week · day · hour · minute · second

⑥ FORMAT — fechas con formato personalizado

format_fechas.sql 
SELECT
    FORMAT(GETDATE(), 'dd/MM/yyyy')           AS FechaCorta,    -- 15/06/2025
    FORMAT(GETDATE(), 'dd/MM/yyyy HH:mm')     AS FechaHora,     -- 15/06/2025 14:32
    FORMAT(GETDATE(), 'dddd d \de MMMM \de yyyy', 'es-AR')
                                                AS FechaLarga,    -- domingo 15 de junio de 2025
    FORMAT(GETDATE(), 'MMMM yyyy', 'es-AR')    AS MesAnio,       -- junio 2025
    FORMAT(GETDATE(), 'yyyyMMdd')             AS FechaISO;      -- 20250615
⚠️
FORMAT() es muy flexible pero más lenta que CONVERT() porque usa el motor de .NET internamente. Para reportes con millones de filas, preferí CONVERT(). Para cantidades normales, FORMAT() es la opción más legible.

Funciones de conversión de tipos

SQL Server es un lenguaje fuertemente tipado: cada columna tiene un tipo de dato definido. A veces es necesario convertir un valor de un tipo a otro para poder operar, comparar o mostrar datos.

Hay dos tipos de conversión: implícita (SQL Server la hace automáticamente cuando los tipos son compatibles) y explícita (vos la indicás con CAST o CONVERT). La conversión explícita es siempre más segura y más clara.

① CAST vs CONVERT

CAST — estándar SQL
-- Sintaxis: CAST(valor AS tipo)
SELECT
  CAST(42 AS VARCHAR(10))   -- '42'
  CAST('3.14' AS DECIMAL(5,2)) -- 3.14
  CAST(GETDATE() AS DATE)    -- solo fecha
  CAST(1 AS BIT)              -- 1 (true)
CONVERT — específico SQL Server
-- Sintaxis: CONVERT(tipo, valor [, estilo])
SELECT
  CONVERT(VARCHAR, 42)         -- '42'
  CONVERT(DATE, '2025-06-15')  -- fecha
  -- Con estilo de formato de fecha:
  CONVERT(VARCHAR, GETDATE(), 103) -- dd/mm/yyyy
  CONVERT(VARCHAR, GETDATE(), 112) -- yyyymmdd

② Estilos de CONVERT para fechas

EstiloFormatoEjemplo
CONVERT(..., 101)mm/dd/yyyy (USA)06/15/2025
CONVERT(..., 103)dd/mm/yyyy (BR/AR)15/06/2025
CONVERT(..., 104)dd.mm.yyyy (DE)15.06.2025
CONVERT(..., 108)hh:mi:ss14:32:00
CONVERT(..., 112)yyyymmdd20250615
CONVERT(..., 120)yyyy-mm-dd hh:mi:ss2025-06-15 14:32:00

③ TRY_CAST / TRY_CONVERT — conversión segura

Si intentás convertir un valor inválido con CAST o CONVERT, SQL Server lanza un error y la consulta falla. TRY_CAST y TRY_CONVERT devuelven NULL en lugar de fallar, lo que permite manejar datos sucios sin interrumpir la consulta.
try_cast.sql 
-- CAST falla si el valor no es convertible
SELECT CAST('abc' AS INT)      -- ❌ Error: conversion failed

-- TRY_CAST devuelve NULL en lugar de error
SELECT TRY_CAST('abc' AS INT)  -- ✓ NULL
SELECT TRY_CAST('42'  AS INT)  -- ✓ 42

-- Uso práctico: filtrar filas con datos inválidos
SELECT ValorImportado
FROM   ImportacionExcel
WHERE  TRY_CAST(ValorImportado AS DECIMAL(10,2)) IS NULL; -- filas con error

④ ISNULL y COALESCE — reemplazar NULLs

isnull_coalesce.sql 
-- ISNULL: si el valor es NULL, devuelve el reemplazo
SELECT
    Nombre,
    ISNULL(Telefono, 'Sin datos')    AS Tel,
    ISNULL(Comision, 0)              AS Comision

FROM Empleados;

-- COALESCE: devuelve el primer valor NO NULL de la lista
-- (estándar SQL, más flexible que ISNULL)
SELECT
    Nombre,
    COALESCE(Celular, Telefono, Email, 'Sin contacto') AS Contacto
FROM Empleados;

-- COALESCE para calcular con NULLs de forma segura
SELECT
    Sueldo + COALESCE(Comision, 0) AS SueldoTotal
FROM Empleados;
💡
Preferí COALESCE sobre ISNULL cuando tenés más de una alternativa o cuando necesitás código portable (es estándar SQL). Ambas hacen lo mismo con un solo argumento de reemplazo.

Funciones numéricas

Funciones para redondear, truncar, obtener valores absolutos, calcular potencias y raíces, y generar números aleatorios.

ROUNDNúmero
Redondea un número a la cantidad de decimales indicada. Si decimales es negativo, redondea a la izquierda del punto.
ROUND(numero, decimales)
ROUND(3.567, 2) → 3.570
ROUND(3.567, 0) → 4.000
ROUND(345, -2) → 300
FLOORNúmero
Devuelve el mayor entero menor o igual al número (redondea hacia abajo).
FLOOR(numero)
FLOOR(3.9) → 3
FLOOR(-3.1) → -4
CEILINGNúmero
Devuelve el menor entero mayor o igual al número (redondea hacia arriba).
CEILING(numero)
CEILING(3.1) → 4
CEILING(-3.9) → -3
ABSNúmero
Devuelve el valor absoluto (siempre positivo) de un número.
ABS(numero)
ABS(-42) → 42
ABS(7) → 7
POWERNúmero
Calcula base elevada a la potencia indicada.
POWER(base, exponente)
POWER(2, 10) → 1024
POWER(3, 3) → 27
SQRTNúmero
Calcula la raíz cuadrada de un número. El número debe ser positivo.
SQRT(numero)
SQRT(144) → 12
SQRT(2) → 1.4142...
RANDNúmero
Genera un número decimal aleatorio entre 0 y 1. Con semilla, el resultado es reproducible.
RAND() / RAND(semilla)
RAND() → 0.7382...
RAND(42) → siempre igual
SIGNNúmero
Devuelve el signo del número: 1 (positivo), 0 (cero), -1 (negativo).
SIGN(numero)
SIGN(100) → 1
SIGN(-5) → -1
SIGN(0) → 0

Caso práctico — cálculo de totales con redondeo

numeros_practico.sql 
SELECT
    Producto,
    PrecioBase,
    -- Precio con IVA redondeado a 2 decimales
    ROUND(PrecioBase * 1.21, 2)          AS PrecioConIVA,

    -- Precio siempre hacia arriba (para no cobrar de menos)
    CEILING(PrecioBase * 1.21)           AS PrecioCeiling,

    -- Diferencia con el precio de lista (valor absoluto)
    ABS(PrecioBase - PrecioLista)          AS Diferencia,

    -- Descuento: si hay pérdida (negativo) → 0
    CASE WHEN PrecioBase - Costo > 0
         THEN ROUND(PrecioBase - Costo, 2)
         ELSE 0
    END                                    AS Ganancia

FROM Productos;
